## Требования  
  
- Linux x86_64  
- `gcc`, `make`  
- `curl`, `git`, `cmake`, `perl`  
- `xterm` — только для команды `make run`  
  
## Сборка  
  
Сначала соберите локальные криптографические зависимости:  
  
```bash  
make deps  
```  
  
Затем соберите сервер и клиент:  
  
```bash  
make  
```  
  
После сборки бинарные файлы будут находиться в каталоге `bin/`:  
  
```text  
bin/server  
bin/client  
```  
  
## Запуск  
  
### Автоматический запуск демо  
  
```bash  
make run  
```  
  
Команда пересобирает проект и открывает три окна `xterm`: сервер, клиент `Alice` и клиент `Bob`.  
  
### Ручной запуск  
  
В одном терминале запустите сервер:  
  
```bash  
./bin/server  
```  
  
В отдельных терминалах запустите клиентов:  
  
```bash  
./bin/client Alice  
./bin/client Bob  
```  
  
Сервер использует порт `5555`, клиент подключается к `127.0.0.1:5555`.  
  
## Использование клиента  
  
После запуска клиент ещё не находится в чате. Сначала выполните регистрацию или вход:  
  
```text  
/register NAME # зарегистрировать пользователя и создать/использовать локальные ключи  
/login NAME # войти с уже существующими локальными ключами  
/help # показать доступные команды  
```  
  
Если имя было передано при запуске клиента, можно использовать короткие команды:  
  
```text  
/register  
/login  
```  
  
После входа обычный текст отправляется в текущую комнату. Для смены комнаты:  
  
```text  
/join ROOM_ID  
```  
  
Пример:  
  
```text  
/register Alice  
Привет!  
/join 2  
Сообщение в комнате 2  
```  
  
## Полезные команды Makefile  
  
```bash  
make deps # собрать OpenSSL и GOST provider  
make # собрать client и server  
make run # запустить демо с сервером и двумя клиентами  
make stop # остановить запущенные client/server  
make clean # удалить объектные файлы и бинарники  
```  
  
## Локальные данные  
  
Проект создаёт служебные каталоги:  
  
- `.deps/` — собранные зависимости  
- `bin/` — бинарники, библиотеки и OpenSSL modules  
- `obj/` — объектные файлы  
- `keys/` — локальные ключи клиентов  
- `ksi/` — серверное хранилище ключей
