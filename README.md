## Требования

- Linux x86_64
    
- `gcc`, `make`
    
- `curl`, `git`, `cmake`, `perl`
    
- `kitty`, `wmctrl` 

## Сборка

Cоберите локальные криптографические зависимости:

```bash
make deps
```

Cоберите сервер и клиент:

```bash
make
```

После сборки бинарные файлы и runtime-зависимости будут находиться в каталоге `bin/`:

```text
bin/server
bin/client
bin/lib/
bin/ossl-modules/
```

`bin/lib/` - локальные библиотеки OpenSSL
`bin/ossl-modules/` - GOST provider.

## Запуск

### Автоматический запуск демо

```bash
make demo
```

Команда пересобирает проект, останавливает ранее запущенные экземпляры и открывает три окна `kitty`:

```text
CHAT_SERVER — сервер
CHAT_ALICE  — клиент Alice
CHAT_BOB    — клиент Bob
```

Окна автоматически размещаются на экране через `wmctrl`.

### Ручной запуск

В одном терминале запустите сервер:

```bash
./bin/server
```

В отдельных терминалах запустите клиентов:

```bash
./bin/client Alice
./bin/client Bob
```

Сервер использует порт `5555`, клиент подключается к `127.0.0.1:5555`.

## Интерфейс клиента

После запуска клиент показывает стартовое меню:

```text
╭────────────────────────────────────────╮
│              secure chat               │
├────────────────────────────────────────┤
│ default name: Alice                    │
│                                        │
│ 1  login as default user               │
│ 2  register default user               │
│ 3  show commands                       │
│                                        │
│ or type: /login NAME, /register NAME   │
╰────────────────────────────────────────╯
```

Если имя было передано при запуске клиента, можно использовать короткие команды:

```text
/login
/register
```

Или выбрать пункт меню:

```text
1 — войти под именем по умолчанию
2 — зарегистрировать имя по умолчанию
3 — показать команды
```

Также можно явно указать имя:

```text
/login NAME
/register NAME
```

## Команды до входа

До входа в чат доступны команды:

```text
/login           войти под именем по умолчанию
/register        зарегистрировать имя по умолчанию
/login NAME      войти под указанным именем
/register NAME   зарегистрировать указанное имя
/help            показать меню команд
```

При регистрации клиент создаёт локальные ключи пользователя. 
При входе используются уже существующие локальные ключи.

## Команды после входа

После успешного входа обычный текст отправляется как сообщение в текущую комнату.

Основные команды:

```text
/join ROOM_ID                    перейти в комнату
/create_room ROOM_ID             создать обычную комнату
/create_room_password ROOM_ID    создать комнату с паролем
/help                            показать меню команд
```

Текущая комната отображается в справке:

```text
╭─ chat commands ────────────────────────╮
│ current room: #1                       │
│                                        │
│ /join ROOM_ID                          │
│ /create_room ROOM_ID                   │
│ /create_room_password ROOM_ID          │
│ /help                                  │
│                                        │
│ plain text sends message to room       │
╰────────────────────────────────────────╯
```

## Комнаты

По умолчанию пользователь входит в комнату `#1`.

Для создания обычной комнаты:

```text
/create_room 2
```

После создания нужно перейти в неё:

```text
/join 2
```

Для создания комнаты с паролем:

```text
/create_room_password 5
```

После этого клиент попросит ввести пароль. Пароль не выводится открытым текстом: при вводе отображаются символы `*`.

Для входа в парольную комнату другой пользователь выполняет:

```text
/join 5
```

После запроса пароля клиент проверяет пароль и при успешной проверке входит в комнату.

## Пример работы

Клиент Alice:

```text
/register
/create_room_password 5
5555
/join 5
hello from Alice
```

Клиент Bob:

```text
/register
/join 5
5555
hello from Bob
```

После входа обоих клиентов сообщения отправляются в комнате `#5`.

## Команды Makefile

```bash
make deps     # собрать OpenSSL и GOST provider
make          # собрать client и server
make demo     # запустить демо с сервером, Alice и Bob
make server   # открыть отдельное окно сервера
make alice    # открыть отдельное окно клиента Alice
make bob      # открыть отдельное окно клиента Bob
make client   # открыть отдельное окно клиента Client
make stop     # остановить запущенные client/server
make clean    # удалить объектные файлы и бинарники
```

## Локальные данные

Создаются служебные каталоги:

- `.deps/` — собранные криптографические зависимости
    
- `bin/` — бинарные файлы, runtime-библиотеки и OpenSSL modules
    
- `obj/` — объектные файлы
    
- `keys/` — локальные ключи клиентов
    
- `ksi/` — серверное хранилище публичных ключей
